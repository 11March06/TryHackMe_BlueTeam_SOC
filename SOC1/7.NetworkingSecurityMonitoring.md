## c. A Network-Overview
Before understanding the security aspects of a Network, it is important to understand what a network is, what components or assets form it, and how they work

<img width="973" height="779" alt="image" src="https://github.com/user-attachments/assets/5f5bcd3f-6fff-4aba-9392-1952156c5057" />
  - A computer network is an organized system of connected devices for communication and data sharing

* User Workstations (Endpoints):
  - Used by employees but are the most common attack entry point
  - Often targeted via phishing or malware
  - Compromise can allow attackers to move inside the network

* File & Database Servers:
  - Store critical business data (documents, customer info, financial data)
  - Major targets for ransomware and data theft

* Application Servers (Web, Email, VPN):
  - Provide essential services for users and customers
  - High-value targets due to internet exposure
  - Attacks include exploits, brute-force, and scanning

* Active Directory (AD) / Authentication Servers:
  - Central system for user identity and access control
  - Key target for attackers to gain full network control
  - Monitoring focuses on login anomalies and privilege misuse
    
* Routers & Switches:
  - Connect devices and networks together
  - If compromised, can allow traffic interception or redirection

* Firewalls / Perimeter Devices:
  - First line of defense between internal network and Internet
  - Control and filter traffic based on rules
  - Provide important logs for detecting attacks early

* Main types of logs

** 🖥️Host-Centric Logs (inside a device)**
Generated by individual machines (PCs, servers, laptops)
Show what happens inside a system
  - Examples:
    - Operating system logs (Windows Event Logs, Linux syslog)
    - Application logs (web servers, databases)
    - Security tools logs (antivirus, EDR)
  
  - What they show:
    - Logins and logouts
    - Process execution
    - File or system activity
  --> Purpose: Understand what happened on a specific machine

      Expansion :
  - Host-centric logs are critical for deep investigation inside a single machine
They help security analysts answer key forensic questions:


🔍 Forensic analysis
Show exactly what an attacker did on a compromised device
Example: files accessed, modified, or deleted

⚙️ Process & execution tracking
Detect malicious processes or scripts (e.g., PowerShell)
Track unauthorized program execution and service changes

👤 User activity monitoring
Track logins, logout times, and user privileges
Helps detect external attacks and insider threats

🦠 Malware impact assessment
Confirm if malware ran on the system
Show changes to registry, files, or running services

👉 Overall idea: Host logs help reconstruct exactly what happened inside a compromised system.

**🌐 Network-Centric Logs (between devices)**
Generated by network devices (firewalls, routers, IDS, proxies, VPNs)
Show traffic between systems
  - Examples:
    - Firewall logs (allowed/blocked connections)
    - IDS/IPS alerts (suspicious traffic)
    - Router flow data (who talks to whom)
    - Proxy logs (web browsing activity)
    - VPN logs (remote access activity)

  - What they show:
    - Source and destination IPs
    - Ports and protocols
    - Connection time and volume
  -- >  Purpose: Understand how traffic moves across the network

🔗 Key idea: Correlation
  - Host logs = what happened inside a device
  - Network logs = who connected to whom
  - Together → give full attack picture (timeline of an incident)

  Expansion : 
  Network-centric logs give a high-level view of network traffic across devices
They help security teams monitor activity at the network perimeter and inside the network
🚨 Early threat detection
Detect attacks before they reach internal systems
Examples: port scans, brute-force attempts, malicious IP connections
🎯 Detect Command & Control (C2)
Identify communication between infected devices and attacker servers
Shows that a system is already compromised
🔄 Track lateral movement
Observe attackers moving between internal systems
Helps understand how an attack spreads inside the network
📤 Detect data exfiltration
Identify large or unusual outbound data transfers
Can indicate data theft or breach
🌐 Provide broad attack context
Shows which systems communicated with each other
Helps reconstruct full attack scope

👉 Overall idea: Network logs show who talks to who, when, and how, helping detect and understand attacks across the whole network.

## e. Network Perimeter
  - The network perimeter is the boundary between a trusted internal network and the untrusted Internet
  - It acts like a main gate or front door controlling all incoming and outgoing traffic
  - It is the first line of defense in cybersecurity

🧱 What makes up the perimeter?
  🔥 Firewalls: filter and control traffic between internal and external networks
  🌐 Routers/Gateways: direct traffic and enforce network rules
  🛡️ DMZ (Demilitarized Zone): isolated zone for public-facing services (web, mail, VPN servers)
  🔐 VPN Gateways: allow secure remote access for employees

⚠️ Why the perimeter is important
  - Attackers usually start their attacks from outside the network
  - Weak or misconfigured perimeters can lead to:
    - Unauthorized access (RDP, SSH, databases exposed)
    - Network scanning and reconnaissance
    - Brute-force login attempts
    - Data exfiltration (stolen data sent out)

🏢 Example (small enterprise network)
  - Firewall separates Internet and internal network
  - Web server is placed in DMZ (public access only)
  - Internal systems (AD, file, database servers) are protected behind firewall
  - Employees outside office use VPN to connect securely

👉 This ensures public services are isolated while internal systems stay protected

🧠 Role of a Security Analyst
  - Monitor firewall logs for allowed/blocked traffic
  - Detect scanning and brute-force attempts
  - Identify unusual outbound traffic (possible malware or data theft)
  - Ensure only necessary services are exposed to the Internet

## f. Network Perimeters : Monitoring and Protecting
  - Perimeter monitoring uses firewalls, IDS/IPS, and WAFs to detect and block attacks
  - It helps security teams detect:
    - Early attacks (port scans, brute-force attempts)
    - Misconfigurations exposing services
    - Outbound suspicious traffic (malware or data theft)

Key Scenarios 
  Port Scanning
    - One IP tries many ports on a system
    - Pattern = repeated requests to different ports
    - Verdict : attacker is mapping open services
  Web attacks (WAF) - Web Application Firewall 
    - Block attacks like : 
      - SQL Injection
      - XSS 
      - Directory Traversal
    - Show attacker actively targeting web application
  VPN Brute Force 
    - Many failed login attempts from same source
    - Occasional suscussful logins may still appear
    - Pattern = repeated FAILED_AUTH events

  * Key takeaways
    * Pattern detection is more impportant than single events
    * Repetition = scanning or brute force
    * IDS/WAF alerts give attack context (type of attack)
    * Perimeter monitoring is the first step in detecting cyber attacks

## g. Perimeter Logs : Investigating the Breach
Incident
  - An attacker starts with reconnaissance (scanning ports) from outside
  - Firewall logs show repeated BLOCK attempts --> port scanning activity
  - The attacker then perfomrs VPN brute-force attack
    - Many failed logins --> eventually successful login
    - Gain internal IP (initial access)
  - After access, attacker performs lateral movement
    - Tries to access other internal machines
    - Targets service like SSH (22), SMB(445), RDP (3389)
  - IDS confirms:
    - SMB exploitation
    - RDP brute-force
    - Internal scanning activity
  - Next stage: C2 (Command & Control) communication
    - Compromised host sends regular traffic to external server
    - Indicates malware is controlled remotely
  - Final stage: Data exfiltration
    - Large outbound traffic (HTTP POST uploads)
    - Sensitive data likely being sent outside
   
* **Attack Chain** (simple)
1. Recon (scan)
2. Brute-force VPN
3. Initial access (VPN success)
4. Lateral movement
5. C2 beaconing
6. Data exfiltration

## b. Network Discovery
  * Attackers perform network discovery to find targets
  * They scan for:
    - IP addresses
    - Open ports
    - Operating systems
    -Service versions (to find vulnerabilities)
Goal: identify weak points to exploit

  * Defenders also perform discovery to:
    - Inventory assets
    - Close unnecessary ports/services
    - Patch vulnerabilities
    - Reduce attack surface

  - Both attackers and legitimate entities (admins, search engines) perform scanning
  - Hard to distinguish good vs malicious activity

* Solutions:
  - Allowlist trusted scanners
  - Use threat intelligence to detect malicious sources
  - Monitor behavior patterns (scanning activity)

* **Services** that attackers scan, other than, IP addresses, ports, and OS version, in order to identify vulnerabilities in a network.

## d. Horizontal vs Vertical Scanning 
  - After identifying hosts, attackers perform port scanning to find open services.

Types of scanning:
<img width="795" height="562" alt="image" src="https://github.com/user-attachments/assets/ebe66e81-5d6e-4365-884e-4150a7d242c9" />
  - Horizontal scan
    - One port across many IP addresses
    - Goal: find which machines expose a specific service
    - Pattern: same source IP, same destination port, multiple destination IPs

  <img width="779" height="606" alt="image" src="https://github.com/user-attachments/assets/562d4322-7f9a-47c6-acd1-d274c20a38e5" />
- Vertical scan
    - One IP address across many ports
    - Goal: identify all services on a target machine
    - Pattern: same source IP, same destination IP, multiple destination ports

   - Mixed scan
    -Combination of horizontal and vertical scanning

* Key idea:
  -  Horizontal = which machines have a service
  - Vertical = which services exist on a machine
 
## e. The Mechanics of Scanning
  - Attackers use different scan techniques to discover hosts and open ports.
  - Types of Scans
    - Ping Sweep (ICMP)
    - Sends ICMP requests to many hosts
    - If a host replies → it is online
    - Easy to detect, often blocked

  - TCP SYN Scan
    -  Uses TCP 3-way handshake (SYN → SYN-ACK → ACK)
    - Scanner sends SYN:
      - SYN-ACK → port is open
      - No response / RST → port is closed
    - More stealthy and harder to detect

  - UDP Scan
    - Sends UDP packets
    - ICMP “port unreachable” → port is closed
    - No response → possibly open (uncertain)
    - Slow and unreliable

Key Idea
    - Different scans leave different log patterns
    - SOC analysts use logs (e.g., firewall, Zeek) to:
      - Identify scan type
      - Detect attackers
      - Reduce false positives by excluding trusted scans
