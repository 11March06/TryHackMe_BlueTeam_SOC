# 1. Pyramid of Pain
## a. Introduction 
- The Pyramid of Pain concept is used in cybersecurity solutions like Cisco Security, SentinelOne, and SOCRadar
- It improves:
    - Cyber Threat Intelligence (CTI)
    - Threat Hunting
    - Incident Response
- Understanding it is important for:
    - Threat Hunters
    -Incident Responders
    - SOC Analysts
- It helps to:
    - Identify types of indicators (IOCs)
    - Measure how difficult it is for attackers to adapt
    - Focus on more effective detection strategies
## b. Hash Values (Trivial)
What is a hash?
  - A fixed-length numeric value that uniquely represents data
  - Generated by a hashing algorithm
Common hashing algorithms
  - MD5
    -128-bit
    -Not secure (collision attacks)
  - SHA-1
    - 160-bit
    - Deprecated due to weaknesses
  - SHA-2 (e.g., SHA-256)
    -256-bit
    -More secure and widely used
Use in cybersecurity
  - Identify malicious or suspicious files
  - Used as Indicators of Compromise (IOCs)
  - Common in malware reports

## c. IP address (Easy)
- IP Address in the Pyramid of Pain
    - Identifies devices on a network
    - Considered a low-level indicator (green) → easy for attackers to change
- Defensive use
    - Can:
      - Block or deny IPs via firewall
      - Detect malicious connections
    - Limitation:
      - Attackers can quickly switch to new IPs
- What is Fast Flux?
    - A technique used by botnets to hide infrastructure
    - A domain:
      - Resolves to multiple IP addresses
      - Changes frequently
**According to Akamai:**
- Used for:
    -phishing
    -malware delivery
    -command & control (C2)

**Key takeaway**
  - IP addresses are weak indicators
  - Blocking IPs is not enough
  - Stronger detection should focus on behavior and TTPs

## d. Domain Names (Simple) 
* Domain Names in the Pyramid of Pain
  - Domains map to IP addresses (e.g., evilcorp.com)
  - Higher than IP → harder for attackers to change
  - Still changeable due to flexible DNS providers

* Why domains matter
  - Attackers need to:
    - Purchase domain
    - Configure DNS
    - But automation makes it easier than expected

* Common attack techniques
  1. Punycode Attack
  - Fake domains that look legitimate
  - Example:
    -fake: adıdas.de
    -real: xn--addas-o4a.de
  * Note : type of attack uses Unicode characters in the domain name to imitate the a known domain - Punycode Attack

  2. URL Shortener Abuse
  - Hide malicious links using:
    - bit.ly, tinyurl.com, etc.
  - Add + to preview the real destination
  
* Detecting malicious domains
  Use:
  - Proxy logs
  - Web server logs

*Analysis using sandbox (e.g. ANY.RUN)
  Important tabs:
  - HTTP Requests - Shows resources retrieved
  - Connections - Shows communication with hosts (possible C2)
  - DNS Requests - Shows queried domains + Useful for detecting malware behavior

* Key takeaway
  - Domains are stronger indicators than IPs
  - Still not perfect
  - Behavioral detection is more effective

## e. Host Artifacts (Annoying)
* Host Artifacts in the Pyramid of Pain (yellow zone)
  - These are traces left by attackers on a compromised system (host)
  - Examples:
     - Modified registry keys
     - Suspicious process execution
     - Dropped or modified files
     - Attack patterns or IOCs

* Why it matters
  - Harder to change than IPs or domains
  - If detected, attackers must:
    - change tools
    - change techniques
  - This increases cost and effort significantly

* Example artifacts
 - Word document spawning malicious processes
 - Execution of suspicious applications
 - Files dropped by malware

* Key takeaway
- Host artifacts are direct evidence on the victim machine
- More valuable than IP/domain indicators
- Represent a higher “pain level” for attackers in the Pyramid of Pain

## f. Network Artifacts
* Network Artifacts in the Pyramid of Pain (yellow zone)
 - Network artifacts are traces left in network traffic by attackers
 - Examples:
   - Suspicious User-Agent strings
   - URI patterns in HTTP requests
   - Command & Control (C2) information
   - HTTP POST request patterns
* User-Agent
  - A header in HTTP requests identifying the client
  - Can reveal browser, tool, or malware used
* How to detect network artifacts
  Tools:
  - Wireshark (PCAP analysis)
  - TShark (command-line analysis)
  - IDS systems like Snort
  Example command : /tshark --Y http.request -T fields -e http.host -e http.user_agent -r analysis_file.pcap/

*Why it matters 
- Harder to change than IPs or domains
- Can be used to:
  - Block suspicious traffic
  - Detect malware behaviour patterns
* Key takeaway
  - Network artifacts help detect attacks through traffic behavior
  - They force attackers to modify tools and techniques
  - Stronger than IP/domain indicators in the Pyramid of Pain

 --> Compare: HOST vs NETWORK
* **Host Artifacts**
- Definition: Traces left on the victim machine (endpoint) by an attacker
- Examples: Malicious files, modified registry keys, suspicious processes, persistence mechanisms
- Detection sources: EDR tools, Windows Event Logs, system logs
- Location: Inside the host system
- Change difficulty: Medium (attackers can replace or modify malware)
- Security value: High compared to IP/domain, helps confirm compromise on a device

* **Network Artifacts**
- Definition: Traces left in network traffic generated by attacker activity
- Examples: User-Agent strings, C2 communication, HTTP POST patterns, URI patterns
- Detection sources: PCAP files, Wireshark, TShark, IDS (Snort/Suricata), proxy logs
- Location: Network traffic between systems
- Change difficulty: Higher than host artifacts (requires changing communication behavior/tools)
- Security value: Very high, helps detect and block malicious traffic patterns across networks

## h. TTPs (Tough) 
* TTPs - the top of the Pyramid of Pain
- TTPs stands for:
  - Tactics
  - Techniques
  - Procedures
- This is the highest level in the Pyramid of Pain
- It represents full attacker behaviour mapped in MITRE ATT&CK

* Example : 
- Phishing
- Persistence
- Lateral movement
- Data exfiltraion
- Pass-the-Hash attacks

* Why TTPs matter
- Detecting TTPs makes it extremely hard for attackers to continue
- Forces them to change entire methods and tools

* Attacker options when detected
- Rebuild tools and retrain
- Abandon the target (most likely option)

* Key Takeaway
- TTPs are the most powerful and painful detection level
- They represent full attacker behaviour, not just indicators
- Primary focus of advanced threat hunting and SOC operations

# 2. Cyber Kill Chain
## a. Introduction 
Following attack phases in this: 
- Reconnaissance
- Weaponization
- Delivery
- Exploitation
- Installation
- Command & Control
- Actions on Objectives

## b. Reconnaissance
🔎 Reconnaissance (Recon)
--> Reconnaissance is the first phase of an attack, where attackers gather information about a target to plan their next steps.

- Good recon → more effective and targeted attacks
- Poor recon → sloppy and likely to fail

🧠 OSINT (Open-Source Intelligence)

OSINT means collecting information from public sources, such as:
- Search engines
- Social media
- News and blogs
- Public records
- WHOIS / technical data

⚙️ Types of Recon
    i. Passive Recon

No direct interaction with the target
Hard to detect
Example: WHOIS lookup, social media research
    ii. Active Recon

Direct interaction with the target
Easier to detect
Example: port scanning, probing services
📧 Email Harvesting

Collecting email addresses to use in attacks like phishing.

🛠️ Common Tools
theHarvester → gathers emails, domains, IPs
Hunter.io → finds emails from domains
OSINT Framework → collection of OSINT tools

## c. Weaponization
⚔️ Weaponization
--> After reconnaissance, the attacker turns collected information into a usable attack payload.
🔑 Key Terms
    - Malware → malicious software that damages or gains access
    - Exploit → code that takes advantage of a vulnerability
    - Payload → the actual malicious code executed on the victim
🧠 What happens in this phase?
The attacker:
    - Creates or buys malware (e.g., from the Dark Web)
    - Combines exploit + payload into a deliverable attack
⚙️ Common Techniques
    - Malicious Microsoft Office files (macros/VBA)
    - Infected USB drives
    - Setting up C2 (Command & Control) servers
    - Installing backdoors
    - Creating phishing templates that look legitimate

## d. Delivery
📦 Delivery
--> Delivery is the phase where the attacker sends the payload to the target.

🧠 What happens in this phase?
The attacker chooses how to deliver malware into the victim’s system.
🚀 Common Methods
Phishing Email
→ Malicious link or attachment (can be spear phishing for specific targets)
USB Drop Attack
→ Infected USB left in public or sent to victims
Watering Hole Attack
→ Compromised websites that victims trust → leads to malware download

## e. Exploitation
💥 Exploitation
Exploitation is when the attacker’s malicious code runs on the target system by taking advantage of a vulnerability.

🧠 What happens in this phase?
The payload is executed
The attacker gains initial access
May escalate privileges or move inside the network
⚙️ Common Techniques
Malicious macros → triggered when opening infected files
Zero-day exploits → unknown, unpatched vulnerabilities
Known CVEs → publicly known but unpatched flaws
🚨 Signs of Exploitation
Unexpected processes running
Registry changes / new services
Suspicious command-line activity

## f. Installation

🛠️ Installation (Persistence)
After gaining access, the attacker installs a backdoor to maintain long-term access to the system.
🧠 What happens in this phase?
Create persistent access
Ensure they can reconnect later even if the system is patched or rebooted
⚙️ Common Techniques
Web shell → malicious script on a web server
Backdoor (e.g., Meterpreter) → remote control of victim machine
Modify Windows services → run malware automatically
Registry Run Keys / Startup Folder → execute malware at login
🕵️ Evasion Technique
Timestomping → modify file timestamps to avoid detection

## g. Command & Control
🌐 Command & Control (C2)
--> C2 is the phase where the attacker establishes communication with the infected system to control it remotely.

🧠 What happens in this phase?
Malware connects to an attacker-controlled server
The infected machine sends regular signals (beaconing)
Attacker can send commands and control the system
⚙️ Common C2 Channels
HTTP/HTTPS (ports 80/443)
→ blends with normal web traffic (harder to detect)
DNS (DNS tunneling)
→ uses DNS requests to communicate secretly
(Older method) IRC → now mostly outdated

## h. Actions on Objectives (Exfiltration)
--> After going through six phases of the attack, Megatron can finally achieve his goals, which means taking action on the original objectives. With hands-on keyboard access, the attacker can achieve the following: 

* Collect the credentials from users.
* Perform privilege escalation (gaining elevated access like domain administrator access from a workstation by exploiting the misconfiguration).
* Internal reconnaissance (for example, an attacker gets to interact with internal software to find its vulnerabilities).
* Lateral movement through the company's environment.
* Collect and exfiltrate sensitive data.
* Deleting the backups and shadow copies. Shadow Copy is a Microsoft technology that can create backup copies, snapshots of computer files, or volumes. 
Overwrite or corrupt data.
